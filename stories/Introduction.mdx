# SuperClaude Genie UI Components

Welcome to the SuperClaude Genie design system documentation. This Storybook showcases all the UI components used in the SuperClaude Genie Interactive Docs application.

## üé® Design Philosophy

Our design system is built on the principles of:

- **Modern Glassmorphism**: Beautiful frosted glass effects with depth and dimension
- **Smooth Animations**: Framer Motion powered interactions
- **Dark Mode First**: Optimized for dark environments with light mode support
- **Accessibility**: WCAG compliant components built with Radix UI
- **Performance**: Optimized for fast rendering and smooth interactions

## üéØ Key Features

### Variants
Most components support multiple variants:
- `default`: Standard appearance
- `glass`: Glassmorphic with backdrop blur
- `glassDark`: Darker glass variant
- `holographic`: Iridescent colors with shimmer
- `chromatic`: Vibrant chromatic aberration
- `magic`: Animated borders with particles

### Sizes
Components typically offer multiple sizes:
- `sm`: Small size for compact layouts
- `default`: Standard size
- `lg`: Large size for emphasis
- `xl`: Extra large for hero sections

### States
All interactive components support:
- Hover effects
- Focus states
- Disabled states
- Loading states (where applicable)
- Error states (for form components)

## üìö Best Practices

1. **Consistency**: Use the same variant across related components
2. **Accessibility**: Always include proper labels and ARIA attributes
3. **Performance**: Use loading states for async operations
4. **Responsive**: Test components across different screen sizes
5. **Dark/Light Mode**: Ensure components work in both themes
